Memulai Dengan AngularJS

Memulai dengan AngularJS
  • Materi
  • Diskusi

Buat file HTML baru dan tempelkan kode berikut ini: 

  
      <!DOCTYPE>
      <htm ng-app>
      <head>
        <title>Hello, Angular</title>
        <script src="https://code.angularjs.org/1.5.8/angular.min.js"></script>
        </head>
      <body ng-init="name='world'">
        <label>Nama</label>
        <input ng-model="name"/>
        <span>Hallo, {{ name }}!</span>
        <p ng-bind="name"></p>
      </body>
      </html>

Saat membuka file dengan browser, Anda akan melihat field input diikuti dengan teks Hello, World!. Mengedit Nilai input akan mengupdate teks secara real-time, tanpa perlu refresh keseluruhan halaman.

Bedah Kode :
  1. Muat framework angular dari Jaringan Pengiriman Konten. 
    <script src="https://code.angularjs.org/1.5.8/angular.min.js"></script>
  2. Tentukan dokumen HTML sebagai aplikasi Angular dengan perintah ng-app 
    <html ng-app>
  3. Inisialisasi variabel nama menggunakan ng-init 
    <body ng-init="name='world'">
    Perhatikan bahwa ng-init hanya boleh digunakan untuk tujuan demonstrasi dan pengujian saja. Saat membangun aplikasi yang sebenarnya, pengendali harus menginisialisasi data.
  4. Bind data dari model ke tampilan pada kontrol HTML. Bind sebuah 
    <input>
    ke nama properti dengan ng-model 
    <input ng-model="name"/>
  5. Tampilkan konten dari model menggunakan kurung kurawal {{}} 
    <span> Halo, {{name}} </span>
  6. Cara lain untuk mengikat nama properti adalah menggunakan ng-bind bukan handlebars "{{}}" 
    <span ng-bind="name"></span>

Tiga langkah terakhir membuat dua arah pengikatan data. Perubahan yang dilakukan pada input update model, yaitu tercermin dalam tampilan Ada perbedaan antara menggunakan setang dan ng-bind. Jika Anda menggunakan setang, Anda mungkin melihat yang sebenarnya Halo, {{name}} saat halaman dimuat sebelum ekspresi terpecahkan (sebelum data dimuat) sedangkan jika Anda menggunakannya ng-bind, itu hanya akan menampilkan data saat nama teratasi. Sebagai alternatif, perintah ng-cloak bisa digunakan untuk mencegah setang dipasang sebelum dikompilasi.
